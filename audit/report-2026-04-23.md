# NeoGen Store — Brand Audit v1.0
**Date:** 2026-04-23
**Audit source:** live DOM via headless Chromium on pve1 (SSH → `node audit/fetch.mjs`)
**Target:** `https://neogen.store/` (homepage, `lang="ar"`, `dir="rtl"`)
**Spec:** `.claude/agents/neogen-brand-guardian.md` (v1.0 extracted 2026-04-22)

## VERDICT
⚠️ **Close-but-drifting — 10 ✅ / 2 ⚠️ / 7 ❌ / 1 ❓**

The color system is largely correct (Electric Blue, canvas, hover, body/secondary type all match). The drift concentrates on **primary-button styling** (wrong font, wrong radius, missing gradient, low-contrast text) and **hero typography** (Reem Kufi at 14px where the spec expects Rakkas display at clamp(60–180px)). A stale `--nx-brass*` naming pattern remains in CSS tokens — the *values* are correct but the *names* reference the deprecated era.

## What next — prioritized

1. **❌ Fix the primary button styling** — biggest functional drift + accessibility failure.
   - Target: `themes/blocksy-child/` (child-theme CSS override) or a new snippet in `plugins/neogen-snippets/`.
   - Restore `background: linear-gradient(180deg, #00D1FF 0%, #0099CC 100%)`, `border: 1px solid #00D1FF`, `color: #050505` (dark text on light accent → ~14:1), `border-radius: 6px` (matches the existing `--theme-button-border-radius` token), `font-family: "JetBrains Mono"` — or if AR-first labels stay, propose adding JetBrains Mono as an override for EN-only buttons and leaving IBM Plex Sans Arabic for AR, then ask before committing a bilingual button-font rule.

2. **❌ Restore a real hero H1 using Rakkas** — currently the homepage has no display-scale heading; the SEO page-title is the only H1.
   - Add a hero block (or override Blocksy's page title) with `font-family: Rakkas`, `font-size: clamp(60px, 8vw, 180px)`, copy = primary tagline `جيل التقنية القادم` (§ Voice).
   - Overlay target: `themes/blocksy-child/front-page.php` or a WP block pattern registered from `plugins/neogen-snippets/`.

3. **❌ Rename `--nx-brass*` tokens to `--nx-accent*`** to flush deprecated lexicon.
   - `--nx-brass` → `--nx-accent` (already exists; dedup)
   - `--nx-brass-light` → `--nx-accent-hover` (value `#66E4FF`)
   - `--nx-brass-glow` → `--nx-accent-glow`
   - `--nx-shadow-brass` → `--nx-shadow-accent`
   - Overlay target: wherever these are declared (likely `themes/blocksy-child/` or a mu-plugin that injects `<style>`). `grep -r "nx-brass" themes/blocksy-child/ mu-plugins/ plugins/` is the entry point.
   - Do NOT just add aliases — remove the old names once everything references the new ones. One user-visible deploy, so bump `VERSION` + `NEOGEN_CUSTOM_VERSION` atomically.

4. **⚠️ Resolve the button radius inconsistency** — the site exposes `--theme-button-border-radius: 6px` and `--nx-pill: 999px`, but the rendered button is `border-radius: 100%`. Decide: pill (999px / 100%) or sharp (6px)? Spec § Known gaps #2 lists this as open — propose `6px` for rectangular CTAs and `999px` only for icon-circle buttons, and ask for sign-off.

5. **⚠️ Audit the stray `#b8bed0`** — grep the same overlay targets for `#b8bed0`; if it's from Blocksy defaults, either tolerate or neutralize via a child-theme override.

6. **❓ Prove font loading end-to-end** — next audit iteration: extend `audit/fetch.mjs` to read `document.fonts` (FontFaceSet) instead of CSSOM `@font-face`; that API returns loaded fonts across origins. Confirm Chakra Petch, Manrope, JetBrains Mono, Rakkas, Major Mono Display are all actually fetched.
